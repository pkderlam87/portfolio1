Post the data from the contact form to WordPress

What was changed:

The form is sent to the WordPress backend. I can access the data inside the “inbound messages” in the flamingo framework. The user receives the same error messages as before, but the success message comes as an API response. I added an image to this message too.

How I changed it:

I got some help from Professor MJ Phillip and from my colleagues who provided me with good sources of research. So I used the Mészáros Róbert article to implement this feature. To handle the file image I used the following article: https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API/Using_Fetch#supplying_request_options. Then, I used the facilities of the Contact Form 7 plugin to adjust the minimum length to the fields. As well, the messages come from as an API response. The API provides with an “error_id” to show the corresponding error message. This time, I’m using switch statements instead the if statements.

Why I did it that way:

The post method is new for me, meaning that I’m still learning what I can do or not. I made some adjustments to the Mészáros Róbert function, but the essence come from the his article. I tried to access the data from the form, but it was possible only with the flamingo plugin, ins this regard, my colleague (James Farmer) helped me. My merit was all about using the Contact Form 7 to handle the success and error messages. I would like to learn more about this matter. Nevertheless, I could make my functions more readable and more maintainable
